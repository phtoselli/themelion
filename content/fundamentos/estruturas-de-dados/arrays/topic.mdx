## O que é

Um **array** é uma estrutura de dados fundamental que armazena elementos em sequência contígua na memória. Cada elemento é acessado por um índice numérico (geralmente começando em 0).

## Por que importa

Arrays resolvem o problema de **organizar múltiplos valores relacionados** sem precisar criar variáveis separadas.

**Sem arrays:** `idade1 = 10`, `idade2 = 15`, `idade3 = 20` → impossível escalar

**Com arrays:** `idades = [10, 15, 20]` → fácil de iterar, buscar e manipular

## Como funciona

Arrays alocam um bloco contíguo de memória. O acesso é **O(1)** porque o endereço de qualquer elemento pode ser calculado diretamente:

```
endereço = base + (índice × tamanho_do_elemento)
```

Essa propriedade torna arrays extremamente rápidos para leitura e escrita por índice.

## Quando usar (e quando não usar)

**Use arrays quando:**
- Você precisa acessar elementos por índice (acesso aleatório rápido)
- O tamanho é conhecido ou previsível
- Você precisa iterar sobre todos os elementos

**Não use arrays quando:**
- Você precisa inserir/remover elementos frequentemente no meio (use linked lists)
- Você precisa buscar elementos por valor específico sem índice (use hash tables)
- O tamanho é altamente dinâmico e imprevisível

## Erros comuns

- **Não validar índices antes de acessar** → causa index out of bounds
- **Confundir tamanho com último índice** → `array[array.length]` é inválido (deve ser `array[array.length - 1]`)
- **Modificar array durante iteração** → pode causar comportamento inesperado
